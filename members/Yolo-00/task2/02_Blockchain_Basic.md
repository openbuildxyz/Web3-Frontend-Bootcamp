## [简答题] 简述区块链的网络结构

```
去中心化网络：
1. 区块（每个区块对应一条数据或交易,包含区块hash、前一个区块hash、时间戳、难度值、交易数量、矿工奖励、交易信息：from、to等）
2. 链（每个区块按照时间顺序组成的链式的数据结构，每个区块内存在关联上个区块的hash值，保证链的完整性）
3. 安全性（采用hash + 签名的方式保证数据的不可篡改）
4. 智能合约（类似函数，可以自动执行，无需人工干预）
5. 奖励机制（俗称挖矿）
6. 节点（类似数据库的表）
7. 全球统一（交易广播到每个节点）
```

## [简答题] 智能合约是什么，有何作用？

```
1. 类似执行函数，自动执行，无需人工干预
2. 不可更改，部署到链上后无法更改
3. 去中心化，分别在每个节点
```

## [简答题] 怎么理解大家常说的 `EVM` 这个词汇？

```
1. 以太坊的虚拟机
2. 能够执行智能合约的代码
3. 类似前端的node环境后端的Java环境，不过EVM是运行在每个节点上的
```

## [分析题] 你对去中心化的理解

```
1. 防垄断
2. 公开、透明，减少了腐败和欺诈的可能性
3. 安全，采用hash的方式确保数据不可篡改
4. 无法人为干预
```

## [分析题] 比较区块链与传统数据库，你的看法？

```
传统数据库：
1. 集中式存储：传统数据库通常是集中式的，由一个或多个集中控制的服务器管理。服务器出现问题，数据就会收到影响或者丢失
2. 表结构：数据以表的形式存储，表由行和列组成，适用于结构化数据的存储和管理。
3. 可修改性：数据可以被插入、更新和删除，操作灵活但缺乏不可篡改性。
4. 中心化

区块链：
1. 分布式账本：区块链是一个分布式账本，数据被存储在多个节点上，每个节点都有完整的数据副本。
2. 链式结构：数据以区块的形式存储，每个区块包含一组交易，并通过加密哈希与前一个区块链接，形成链式结构。
3. 不可篡改性：一旦数据被记录在区块链上，就无法被修改或删除，保证了数据的不可篡改性和审计追踪性。
4. 去中心化
```
